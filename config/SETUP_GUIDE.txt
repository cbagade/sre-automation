OPERATIONAL SIGNALS - SETUP GUIDE
===================================

This guide explains how to configure the Operational Signals page to fetch data from your git repository.

## CONFIGURATION OVERVIEW

The system fetches daily operational issues from a git repository organized by:
- Region (e.g., Tokyo, Dallas, Paris)
- Date (YYYY-MM-DD format)
- Timeslot (4 slots per day: 00_hr-06_hr, 06_hr-12_hr, 12_hr-18_hr, 18_hr-00_hr)

## STEP 1: CONFIGURE GIT REPOSITORY

You have TWO options to configure the git repository:

### Option A: Using Environment Variables (.env file) - RECOMMENDED

1. Copy .env.example to .env:
   cp .env.example .env

2. Edit .env and set these variables:

   # Git Repository URL
   OPS_SIGNALS_GIT_BASE_URL=https://raw.githubusercontent.com
   OPS_SIGNALS_GIT_OWNER=your-organization
   OPS_SIGNALS_GIT_REPO=operational-signals-repo
   OPS_SIGNALS_GIT_BRANCH=main

   # For Private Repositories (Optional)
   OPS_SIGNALS_GIT_TOKEN=ghp_your_github_token_here
   OPS_SIGNALS_GIT_USE_AUTH=true

### Option B: Direct Configuration (config/operational_signals_config.py)

Edit config/operational_signals_config.py and update the GIT_CONFIG dictionary:

   GIT_CONFIG = {
       "base_url": "https://raw.githubusercontent.com",
       "repo_owner": "your-organization",
       "repo_name": "operational-signals-repo",
       "branch": "main",
       "token": "your_token_here",  # Optional
       "use_auth": True,  # Set to True if using token
   }

## STEP 2: REPOSITORY STRUCTURE

Your git repository should have this structure:

data/
├── Tokyo/
│   ├── 2026-06-16/
│   │   ├── 00_hr-06_hr/
│   │   │   ├── active_critical_immediate_alerts.json
│   │   │   ├── clusters_needing_attention.json
│   │   │   ├── bgp_down.json
│   │   │   └── ... (other JSON files)
│   │   ├── 06_hr-12_hr/
│   │   ├── 12_hr-18_hr/
│   │   └── 18_hr-00_hr/
│   └── 2026-06-17/
├── Dallas/
├── Paris/
└── ... (other regions)

## STEP 3: AUTHENTICATION (For Private Repositories)

### GitHub:
1. Go to Settings > Developer settings > Personal access tokens
2. Generate new token (classic)
3. Select scope: 'repo' (Full control of private repositories)
4. Copy the token and add to .env file

### GitLab:
1. Go to User Settings > Access Tokens
2. Create token with 'read_repository' scope
3. Copy the token and add to .env file

### Bitbucket:
1. Go to Personal settings > App passwords
2. Create app password with 'repository read' permission
3. Copy the password and add to .env file

## STEP 4: CONFIGURE REGIONS

Edit config/operational_signals_config.py and update AVAILABLE_REGIONS:

AVAILABLE_REGIONS = [
    "Tokyo",
    "Dallas",
    "Paris",
    # Add your regions here
]

## STEP 5: CONFIGURE JSON FILES

Edit config/operational_signals_config.py and update JSON_FILES list:

JSON_FILES = [
    "active_critical_immediate_alerts.json",
    "clusters_needing_attention.json",
    # Add your JSON files here
]

## STEP 6: JSON FILE FORMAT

Each JSON file should contain an "issues" array. See config/sample_data_structure.json for examples.

Example:
{
  "issues": [
    {
      "id": "issue-001",
      "title": "Issue description",
      "severity": "critical",
      "component": "network",
      "timestamp": "2026-06-16T03:45:00Z"
    }
  ]
}

## STEP 7: TEST THE CONFIGURATION

1. Start the dashboard:
   streamlit run dashboard/app.py

2. Navigate to "Operational Signals" page
3. Select a date and region
4. Click "Load Operational Signals"

## TROUBLESHOOTING

### "No data available" error:
- Check if the git repository URL is correct
- Verify the repository structure matches the expected format
- Check if data exists for the selected date and region

### Authentication errors (401/403):
- Verify your token is correct
- Check token permissions/scopes
- Ensure OPS_SIGNALS_GIT_USE_AUTH=true in .env

### "File not found" errors:
- Verify JSON file names in config match actual files in repository
- Check the timeslot directory exists for the selected date

## CACHE MANAGEMENT

The system caches fetched data in: data/cache/operational_signals/

To clear cache:
- Delete files in data/cache/operational_signals/
- Or use the agent's clear_cache() method

Cache files are automatically excluded from git (see .gitignore)
